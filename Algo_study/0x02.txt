[Vector]
vector는 가변 배열 생성 가능.
이걸 함수로 넘길 때 기존 배열과 같이 주소값을 넘기는게 아닌 복사된 배열을 넘김 즉 특정 함수로 배열의 값을 바꿀 수는 없음.
그리고 함수로 복사하는 과정에서 만약 N개의 인덱스를 갖는 배열을 넘긴다면 O(N)의 시간복잡도가 소요됨. 즉, 복사하는 과정에서도 연산처리가 이루어짐.
> 이걸 없애려면 vector<int>& v 이런식으로 참조자를 이용해서 매개변수를 받아주면 됨.
>> 선언 방법: vector<자료형> 변수명 > 가변벡터
		  vector<자료형> 변수명(숫자) > 벡터 크기 지정, 0으로 자동 초기화 / 변수명(크기, 숫자) 하면 vector 전부 숫자로 초기화
		  vector<자료형> 변수명 = {변수1, 변수2, 변수3}
		  vector<vector<자료형>> 변수명 : 2차원 배열 생성
>> vector의 Iterators : v.begin() > 벡터 시작점의 주소 값 반환
			     v.end() > 벡터 끝부분 +1의 주소값 반환(벡터 크기 위치에서의 주소값 반환)
			     v.front() > 벡터 첫번째 요소 접근
			     v.back() > 벡터 마지막 요소 접근
			     v.at(i) > 벡터의 i번째 요소 접근
			     v[i] > v.at()이랑 큰 차이 없음, 근데 얘는 벡터 범위를 벗어나도 일단 값을 불러옴.
			     v.push_back(i) > 벡터의 마지막 부분에 새로운 요소 추가 
			     v.pop_back() > 벡터의 마지막 부분 제거
                             v.insert(삽입할 위치의 주소 값, 변수값) > 사용자가 원하는 위치에 요소 삽입
			     v.erase(삭제할 위치) > 사용자가 원하는 위치의 요소 삭제
 
***근데 vector가 insert나 erase 시 데이터를 하나하나씩 이동시키기 때문에 시간초과나 메모리 오버헤드가 발생할 수 있어서 삽입과 삭제가 빈번하게 일어날 경우 vetor보단 list나 deque를 사용하는게 좋다.

[sort]
정렬 시 사용하는 stl > #include <algorithm>에 stl이 들어있다.
기본 오름차순 정렬이다.(마지막 인자 사용하지 않을 경우) / 내림차순 정렬을 하려면 마지막 인자를 사용하여야 한다.
이거 사용하면 시간 복잡도가 최대 nlogn이다.
함수의 구성: sort(배열의 시작주소, 배열의 마지막 주소+1(배열의 크기이다)); // 오름차순의 경우
		ex) sort(arr, arr+10) // 배열 크기 10인 경우, sort(v.begin(),v.end()); // 벡터의 경우
		sort(배열의 시작주소, 배열의 크기, desc(비교 함수의 주소)) // 내림차순의 경우
>> 내림차순의 경우 desc라는 함수를 만들어서 두 수를 입력받고 a>b의 값을 리턴하는 방법으로 내림차순을 만들면 된다.
	이때 함수의 자료형은 bool 형이어야 한다. 즉 return 값을 반환하는 함수여야 함.
	vector 내림차순은 이렇게 >> sort(v.rbegin(), v.rend());
>> 다른 정렬의 경우 desc 함수에서 경우를 나누어서 true, false로 구분해서 저장해주는 방법으로 정렬하면 된다.
stable_sort는 처음 입력된 원소들 중 같은 값을 갖는 원소를 정렬하는 과정에서 처음 입력에서 바뀌지 않는 정렬을 의미한다. >> sort와 같은 방법으로 사용하면 된다.

**** sort함수에서 begin 주소와 end 주소는 반열린 구간이다. 그러므로 [v.begin(), v.end())의 구조를 갖는다. 그러므로 만약 배열 0~3까지의 정렬을 필요로 한다면 주소를 0~4까지 지정해주면 0~3까지의 정렬이 이루어진다.

[string]
이건 기존 c언어를 사용하는 과정에서 가변 문자열을 사용하지 못해 메모리 오류가 많이 뜨는 것들을 한번에 잡아줄 수 있는 유용한 툴이다.
> 기본 vector와 선언 방법 및 사용 방법이 거의 일치함을 알 수 있다.
* 선언 방법: string s; / string s = "문자열"; / string s("문자열"); >> 요정도만 알면 될듯
* 요소 접근: vector와 동일
s[인덱스] > vector는 v[인덱스]인 방면 이게 다르다.(범위 검사 안하는건 똑같다.)
s.at[인덱스] > 범위 검사 해줌. 이게 가장 안정적인 요소 접근 방식인 것 같다.
s.begin(), s.end()는 vector와 같다.
s.clear() > 전부 삭제
s.insert(인덱스, "문자 혹은 문자열"); > 동일하다 >> insert할 시 공백문자가 하나 추가로 마지막에 들어간다.
s.erase(시작 인덱스, 끝 인덱스+1) > sort함수와 같이 반열린 구간이다. (C++은 전부 이런식인듯?)
s.size() > 문자열의 길이를 알고 싶을 때 사용하면 됨.
문자열 입력 받을 때: getline(cin, s); 이런식으로 한줄을 string s에 한번에 받아주면 된다.
일단 요정도만 알아도 다 커버 가능할 듯
