<배열>
메모리상 원소를 연속하게 배치한 자료구조

[배열의 성질]
1. O(1)에 k번째 원소를 확인/변경 가능
2. 추가적으로 소모되는 메모리량(overhead)이 거의 없음
3. Cache hit rate가 높음
4. 메모리상에 연속한 구간을 잡아야 해서 할당에 제약이 걸림

[빅오 시간]
임의의 원소를 확인/변경 > O(1)
첫 원소를 추가할 경우 > O(1)
마지막 원소를 추가할 경우 > O(1)
임의의 자리에 윈소를 추가/제거 > O(N) 

**배열 특정 인덱스 추가/ 제거 시 항상 추가되는 끝공간, 제거되는 공간과 같이 빈 공간을 기준으로 코드를 작성한다.**

**배열 초기화 시 'fill 함수' 사용하여 전부 초기화 한다.**
ex) int a[21]; 인 경우
fill(첫 배열 주소, 마지막 배열 주소+1,초기화 값);
fill(a,a+21,0);

if 2차원 배열의 경우(int b[21][21])
for(int i = 0;i<21;i++)
fill(b[i],b[i]+21,0);

**fill 함수의 경우 [first,last)의 구조를 갖는다. 그렇기에 배열의 이름 + ' ' > 요 안에 들어가는 숫자가 배열의 인덱스 범위라고 생각하면 됨.

<vector>
for문 돌릴 때 vector원소에서 무언가를 빼서 범위를 지정할 때 -1이런거를 넣으면 unsigned int형으로 사이즈 값이 자동 형변환이 되기 때문에 오버플로우가 발생하는 문제가 발생할 수 있다.

**range-based for loop**
이게 ㅈㄴ 편한듯
vector<int> v1 = {1,2,3,4,5,6};
for(int e:v1) >> 이거 사용하면 배열, 문자열, 혹은 백터 등이 전부 e라는 변수로 복사본으로 들어가면서 크기만큼 for문이 돌아감.
만약 for(int &e:v1) >> 이렇게 하면 원본을 보내준거라 실제 값이 들어가고, 이로 인해 배열, 문자열, 혹은 벡터 값을 실제로 참조하여 변경 가능.

**auto 사용법**
이건 자료형을 자동으로 설정해주는 자료형? 이다.
그래서 auto a = 5; 를 입력하면 자동으로 정수형 변수로 만들어줌.
근데 auto arr = {1, 6.2}; 이런식으로 헷갈리게 작성하면 오류 뜨고
함수의 매개변수로 사용하면 안되고, class나 구조체로 사용할 수 없음.

**문자 비교**
특정 문자열에서 문자 하나하나 비교시 for문을 돌리게 되면 문자의 길이만큼 제곱되는 시간 소요가 듬.
>>근데 이걸 해결할 수 있음
어케? 공간을 a~z까지 마련해두고(배열) 문자 하나하나를 배열에 저장해주면 됨. ㄷㄷ

<고찰>
- 문자열이든 숫자든 탐색이 존재할 때 모든 경우의 수를 찾아야 하는 것이 아니면 for문 한개를 줄일 수 있는 확률이 높음. 특히 특정 문자 혹은 숫자를 골라내거나 중복이 되는 부분들을 걸러내는 작업이 필요한 경우, 문자는 아스키 코드를 이용한 판단과 숫자는 bool변수를 이용한 visit 코드 구현을 떠올리는 것이 좋을 듯(한번 방문한 것만 체크하면 중복조합들을 순열 처리해 줄 수 있음)
