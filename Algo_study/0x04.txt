<연결리스트>
배열과 연결리스트는 비슷한 구조를 갖지만 연결리스트는 배열과 같이 연속된 데이터 구조를 갖는게 아니라 여러 곳에 흩어져 있는 데이터의 주소값들을 이전 인덱스가 보유하여 연결된 구조를 이루는 것을 의미한다.

[종류]
- 단일 연결 리스트: 각 원소가 자신의 다음 원소의 주소를 보유
- 이중 연결 리스트: 각 원소가 자신의 이전 원소와 다음원소의 주소를 둘다 보유
- 원형 연결 리스트: 끝 원소와 첫 원소가 서로 연결된 구조

[배열 VS 연결 리스트]
K번째 원소의 접근: O(1) / O(k)
임의 위치에 원소 추가/제거: O(N) / O(1)
(단, 임의의 위치에 원소를 추가하는 과정에서 각 원소의 주소값을 전부 아는 경우만 이에 해당됨. 모르면 그 원소까지 찾아가서 주소값을 찾고 나서 원소를 넣거나 빼야 하기에 O(k)의 시간이 소모됨)
원소 추가 및 제거 시 연결리스트는 단순히 주소값만 추가 및 제거해주면 된다. 
메모리 상의 배치: 연속 / 불연속
추가적으로 필요한 공간: - / O(N)

>> 연결리스트 사용 상황: ex) 메모장 > 특정 구간에 텍스트 작성 및 지움 등을 반복하여 원소를 추가 및 제거하는 상황이 많기 때문.
** 연결리스트의 경우 배열과 다르게 임의의 위치에 원소를 추가/제거 하는 경우가 많은 상황에서 사용해야 한다. 시간 단축용!!**

>> 연결 리스트 사용 시 list stl을 사용하면 좋다. 다만, stl이 허용되지 않는 코테라면 직접 연결리스트를 구현해야 함.


[STL list 예시]
int main(){
 list<int> L = {1,2}; //list에 초기값 1,2를 추가한다.
 list<int>::iterator t = L.begin(); // t는 list의 시작주소를 전달함.
 L.push_front(10); // 10,1,2
 cout << *t <<'\n'; // t 가 가리키는 곳? list의 시작주소 였던 1임!!
 L.push_back(5); // 10,1,2,5
 L.insert(t,6); // t가 가리키는 곳 앞에 6을 추가함 >> 10,6,1,2,5
 t++; // t는 이제 2 위치의 주소값을 가짐
 t = L.erase(t); // t가 가리키는 값을 제거, 그 다음 원소인 5의 위치를 반환
 cout << *t << '\n'; // 5

(최종 list 출력 시)
for(auto i:L) cout << i<< ' ';

or range-based for loop를 모른다?
for(list<int>::iterator it = L.begin();it<L.end();it++)
cout << *it << ' '; 

** 주요점: insert 시 지정한 주소값 앞에 값이 입력되고 반환X, erase 시 지정한 주소값이 가리키는 값을 제거하고 그 다음 원소의 주소값을 반환한다.

******매우 중요!!!
오른쪽 혹은 왼쪽으로 list 주소 이동 시 list든 뭐든 그냥 container stl 혹은 배열을 사용하는 과정에서는 끝점에서의 예외처리가 필수임!!!
왼쪽으로 이동시에는 주소가 begin()이 아닐 경우, 오른쪽으로 이동시에는 주소가 end()가 아닐 경우만 이동 가능하도록 해야함!

[손코딩 문제]
1. 원형 연결 리스트 내의 임의의 노드 하나가 주어졌을 때 해당 list의 길이를 효율적으로 구하는 방법?
 --> 동일한 노드가 나올 때 까지 계속 다음 노드로 이동한다. 공간 복잡도: O(1), 시간복잡도: O(N)

2. 중간에 만나는 두 연결리스트의 시작점들이 주어졌을 때 만나는 지점을 구하는 방법?
 --> 일단 두 시작점 각각에 대해 끝까지 진행시켜서 각각의 길이를 구함. 이후 다시 두 시작점으로 돌아와서 더 긴 쪽을 둘의 차이 만큼 앞으로 먼저 이동시켜놓고 두 시작점이 만날 때 까지 두 시작점을 동시에 한칸씩 전진시키면 된다. 공간복잡도: O(N), 시간 복잡도: O(A+B)

3. 주어진 연결리스트 안에 사이클이 있는지 판단하라
 --> Floyd's cycle-finding algorithm을 이용, 공간복잡도 O(1), 시간복잡도 O(N)으로 처리 가능
