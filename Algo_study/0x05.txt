<스택>
-FILO: First In Last Out / 프링글스 통과 같이 입구가 하나여서 나중에 들어간 놈부터 빼낼 수 있다.

[스텍의 성질]
- 원소의 추가: O(1)
- 원소의 제거: O(1)
- 제일 상단의 원소 확인: O(1)
- 제일 상단이 아닌 나머지 원소들의 확인/ 변경은 원칙적으로 불가능하다.
  (단, 배열로 스택을 구현할 시는 가능함. 그러나 스택 자료구조상 제일 상단을 제외한 나머지 원소들은 알 필요가 없다!!)

*구현은 배열이 쉽다.
> push, pop, top 이렇게 세개만 구현하면 끝남 ㅋㅋㅋ 개날먹임. 그래도 STL을 사용하자!

[STL stack]
stack<int> S; // 스텍의 선언
S.push(10); // 10
S.push(20); // 10 20
S.push(30); // 10 20 30
cout << S.size << '\n'; // 3, 배열 상 NULL부분의 인덱스를 출력한다.
if(S.empty()) cout <<"S is empty\n"; //empty는 bool 변수를 출력한다. empty면, 아니면 0
else cout <<"S is not empty\n";
S.pop(); // 10 20 
cout << S.top() << '\n'; // 20
S.pop(); // 10
cout << S.top() << '\n'; // 10
S.pop(); // -
if(S.empty()) cout << "S is empty\n"; // 출력 됨
cout << S.top() << '\n'; // runtime error 발생

즉, stack에서 주의해야 하는 점은 하나밖에 없다.
stack이 empty인 순간에 top이나 pop을 호출하지 않아야 한다. 이건 if 문으로 항상 empty를 체크하고 호출하도록 해야한다.
*STL stack 사용: pop, push, top, empty, size 요정도...? 더 필요한건 나중에 문제풀면서 찾기
(pop 시 해당 데이터가 반환되는 구조가 아님, 이건 top을 이용해서 먼저 확인해야 함)

** 배열, 벡터, 리스트의 경우 원소를 추가하고 제거하는 과정에서 인덱스 범위를 넘어가지 않는 것을 체크하는 조건이 무조건 있어야 하고,
스택의 경우 pop or top을 사용하는 과정에서 스택이 empty가 아닌지 체크하는 조건이 무조건 들어가야 한다.

*** cin도 문자열을 받는 과정에서 쓸 수 있다. but 띄어쓰기 하면 문자를 다 받는 거
     getline은 한줄의 문자열을 입력받을 때 사용한다. 띄어쓰기 해도 엔터를 치지 않으면 띄어쓰기 포함 한줄을 받는다!

!!! 스택을 사용하는 문제 유형: 최근에 사용한 무언가에 대한 문제, 배열의 마지막 인덱스만 사용하는 문제.
!!! 변화하는 STL은 반복문에 넣을 때 주의한다.
  ex) size와 같은경우 pop을 하게되면 점점 줄어드는데 이게 for문의 조건으로 들어가게 되면 기존 사이즈가 점점 작아져서 겨론적으로 기존 사이즈보다 2배 적게 돌아 논리오류가 발생한다!!
