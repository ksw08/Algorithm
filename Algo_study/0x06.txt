<큐>
-FIFO: First In First Out / 공항에서 입국수속 하듯이 먼저온 놈이 먼저 나간다.

[큐의 성질]
- 원소의 추가: O(1)
- 원소의 제거: O(1)
- 제일 앞/뒤의 원소 확인: O(1)
- 제일 앞/뒤가 아닌 나머지 원소들의 확인/변경이 원칙적으로 불가능
  (단, 큐를 배열로 직접 만들 경우에는 가능함. STL에서는 불가능)

큐를 만들 때는 배열로 만드는 것이 편함.
큐는 크게 선형 큐와 원형 큐가 있는데,
선형 큐의 경우에는 배열의 크기가 정해져 있고 처음에는 head와 tail이 같은 인덱스를 갖고 있지만 push할 때마다 tail의 값이 증가하고 pop할 때마다 head가 증가하는 구조임.
단, 선형 큐의 경우 입력의 개수가 기존 배열의 크기보다 크다고 할 경우 데이터를 전부 push할 수 없는 문제가 있음!
그래서 지속적인 데이터 입력이 필요한 경우, 원형큐를 구현하는 것이 좋음.
원형 큐의 경우, 선형 큐에서 tail이나 head가 마지막 인덱스에 도달할 경우 다시 처음 인덱스로 보내주는 구조로 변경해서 사용하면 됨.
(근데, 코테의 경우 대부분 입력의 개수가 정해져 있기에 공간복잡도가 초과되지 않는 이상 입력 개수보다 큰 배열을 만들어서 사용하면 선형 큐를 사용해도 문제가 없음!)
** tail-head를 하면 queue의 길이를 알 수 있고, pop 시 head만 1증가시켜주면 된다. 굳이 배열을 지울 필요는 없다.

[STL queue]
queue<int> q; // queue의 선언
q.push(10); //10
q.push(20); //10 20
q.push(30); //10 20 30
cout << q.size() << '\n'; // 3
if(q.empty()) cout << "Q is empty\n"; // empty면 1 반환
else cout << "Q is not empty\n";
q.pop(); // 반환자 없음. / 
cout << q.front() << '\n'; // 20
cout << q.back() << '\n'; // 30
q.push(40); // 20 30 40
q.pop(); // 30 40 
cout << q.front() << '\n';

** 큐가 empty 상태인데 front, back, pop을 호출하면 런타임 에러가 발생할 수 있음. 저 세개에는 조건 꼭 걸어놓아야 함.
