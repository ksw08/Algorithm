<재귀>
하나의 함수에서 자기 자신을 다시 호출해 작업을 수행하는 알고리즘

재귀를 다룰 때에는 절자적 사고 대신 귀납적 사고가 가능해야 한다.
>> 예시: 도미노를 여러개 놓고 맨 앞의 놓인 도미노를 쓰러트리면 나머지 도미노들도 쓰러진다.
     why?  절차적 사고: 1 → 2 → 3 → 4 → 5 →.... 순으로 쓰러지니까.
             귀납적 사고: k번째 도미노를 쓰러뜨리면 k+1번째 도미노가 쓰러진다. 그러므로 모든 도미노가 쓰러진다.

[다음은 코드 예시]
void func1(int n) {
 	if (n == 0)return;
 	cout << n << ' ';
 	func1(n - 1);
}

절차적 사고: func1(n) → n출력 → func1(n-1) → n-1 출력 ... 1출력
귀납적 사고: func1(1) → 1출력
 		func1(k) → k k-1 k-2...1 출력하면
 		func1(k+1)  → k+1 k k-1....1 출력한다.

[코드 예시2]
int func2(int n) {
	if (n == 0) return 0;
	return n + func2(n - 1);
}

귀납적 사고:  func2(1) → 1 출력
                  func2(k) → k+k-1+k-2+...+1 출력
		  func2(k+1) → k+1 + k+k-1+k-2+...+1 출력

[재귀 함수의 조건]
1. 특정 입력에 대해서는 자기자신을 호출하지 않고 종료되어야 함.(Base condition)
2. 모든 입력은 base condition 으로 수렴해야 함.
>> 이 두개가 무조건 지켜지지 않으면 런타임 에러가 발생할 수 있음

[재귀에 대한 정보]
1. 함수의 인자로 어떤 것을 받고 어디까지 계산한 후 자기자신에게 넘겨줄지 명확하게 정해야 함.
2. 모든 재귀함수는 반복문만으로 동일한 동작을 하는 함수를 만들 수 있음.
3. 재귀는 반복문으로 구현했을 때에 비해 코드가 간결하지만 메모리/시간에서는 손해를 봄.
4. 하나의 재귀 함수 내에서 자기자신을 여러번 호출하게 되면 상당히 비효율적임. >> 같은 연산을 여러번 반복하게 되기 때문.
   >> 그래서 여러번 호출하게 될 시 N 제곱배의 시간 복잡도가 발생함.
5. 재귀함수가 자기 자신을 호출할 때 스택 영역에 계속 누적이 됨. >> 그래서 함수가 너무 여러번 호출하게 되면 스텍이 터지는 경우가 있음(메모리 오버) 
+ 추가적으로 지역 변수도 스택 메모리에 저장된다. 대부분의 스택 메모리는 1MB 정도 할당이 되어 있으니 큰 배열들은 모두 전역 변수로 선언하는게 좋고 작은 배열 혹은 변수들만 지역 변수로 운용해야 함.
